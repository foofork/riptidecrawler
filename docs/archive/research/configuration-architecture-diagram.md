## Example: Complete Configuration Flow

```rust
// main.rs
fn main() -> Result<()> {
    // 1. Load .env (development)
    dotenvy::dotenv().ok();

    // 2. Load configuration
    let config = RiptideConfig::load()?;
    //    â†“
    //    Loads: config/default.toml
    //    Loads: config/production.toml (if RUST_ENV=production)
    //    Loads: config/local.toml (if exists)
    //    Applies: RIPTIDE_* environment variables

    // 3. Validate
    config.validate()?;
    //    â†“
    //    Checks all required fields
    //    Validates ranges and formats
    //    Ensures secrets are present

    // 4. Load secrets
    let secrets = Secrets::from_env()?;
    //    â†“
    //    Reads: ANTHROPIC_API_KEY
    //    Reads: OPENAI_API_KEY
    //    Fails fast if missing

    // 5. Initialize services
    let server = ServerBuilder::new()
        .with_config(&config.server)
        .with_secrets(&secrets)
        .build()?;

    // 6. Start application
    server.run().await
}

```
Development:
Developer â†’ Edit config/local.toml â†’ Hot-reload (optional)

Staging/Production:
1. Edit config/production.toml
2. Create PR with changes
3. Code review
4. Merge to main
5. CI/CD builds new image
6. Deploy to staging
7. Validate configuration
8. Deploy to production
9. Monitor for issues

Environment-Specific Override:
kubectl set env deployment/riptide \
  RIPTIDE_SERVER_PORT=9000

Secret Rotation:
1. Generate new secret in secret manager
2. Update secret reference
3. Restart pods (rolling update)
4. Old secret deprecated
5. Old secret deleted after grace period
